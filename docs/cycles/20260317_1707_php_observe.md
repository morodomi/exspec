## Implementation Notes

### Goal

PHP observe を実装し、exspec の 4 言語全対応を完了する。PSR-4 namespace-to-file mapping の convention fallback により、composer.json 不要で高精度なテスト-コードマッピングを実現する。

### Background

Phase 9 の最終言語。9a (trait 抽出), 9b (Python), 9c (Rust) 完了済み。PHP 固有課題は PSR-4 namespace-to-file mapping。ROADMAP 方針に従い convention fallback を先行し、composer.json パースは精度不足時のみ追加する段階的アプローチ。

### Design Approach

**stem helpers**

| Input | test_stem | production_stem |
|-------|-----------|-----------------|
| `tests/UserTest.php` | `Some("User")` | `None` |
| `tests/user_test.php` | `Some("user")` | `None` |
| `tests/Unit/OrderServiceTest.php` | `Some("OrderService")` | `None` |
| `src/User.php` | `None` | `Some("User")` |
| `src/Models/User.php` | `None` | `Some("User")` |

**Helper filtering**

| Pattern | Reason |
|---------|--------|
| `TestCase.php` | Base test class |
| `*Factory.php` | Laravel factory |
| `Abstract*.php` in tests/ | Abstract base test |
| `tests/` 配下で test_stem=None のファイル | test helper |
| `Trait*.php` in tests/ | test traits |
| `Kernel.php` | Framework bootstrap |
| `bootstrap.php`, `bootstrap/*.php` | Test bootstrap |

**PSR-4 convention fallback**

1. namespace path から最後のセグメントを symbol、残りを module path とする
2. `App\Models\User` → module=`App/Models`, symbol=`User`
3. scan_root 配下で `Models/User.php` を探索 (common prefixes: `src/`, `app/`, `lib/`, root)
4. 外部パッケージ判定: vendor/ 配下 or 既知フレームワーク namespace (PHPUnit, Illuminate, Symfony, Doctrine 等)

**Barrel**: PHP にはバレル相当パターンなし。`index_file_names()`: `&[]`、`extract_barrel_re_exports()`: 常に `Vec::new()`

**Production function extraction**

- `function` declarations (top-level)
- `class` + `method_declaration` (class methods)
- `public` → is_exported=true、`protected`/`private` → is_exported=false

**Route extraction**: 対象外 (routes は空 Vec)
